# Vibe Coding 提示词库

## 项目启动类

### 生成 PRD/GDD

```
我想做一个 [一句话描述]。

请帮我创建一个简单的产品需求文档（PRD），包含：
1. 产品概述
2. 目标用户
3. 核心功能（P0/P1/P2 分级）
4. 非目标（明确不做什么）
5. 技术约束

保持简洁，不要过度设计。
```

### 推荐技术栈

```
基于以下需求，推荐最简单但足够强大的技术栈：

[粘贴 PRD/GDD]

要求：
1. 挑战自己，提出最简方案
2. 优先考虑学习成本低的技术
3. 社区活跃，文档完善
4. 说明为什么选择这个技术栈
```

### 生成实施计划

```
基于以下文档，创建详细的实施计划：

[PRD/GDD]
[技术栈]

要求：
- 步骤要小而具体
- 每步必须有验证测试
- 只写指令，不写代码
- 先完成核心功能，细节后补
```

## 开发执行类

### 开始新步骤

```
读取 /memory-bank 中的所有文档。
查看 progress.md 了解已完成的工作。
执行实施计划的 Step [X]。

规则：
1. 我会运行测试验证
2. 测试通过前不要开始下一步
3. 完成后更新 progress.md
4. 有架构变更时更新 architecture.md
```

### 澄清需求

```
读取 /memory-bank 中的所有文档。
implementation-plan.md 是否 100% 清晰？
列出你的所有问题，我来回答。
```

### 代码审查

```
审查以下代码，检查：
1. 是否符合项目规范
2. 是否有潜在 bug
3. 是否有性能问题
4. 是否有安全隐患
5. 代码是否足够简洁

[粘贴代码]
```

## 调试类

### 错误排查

```
## 预期行为
[描述应该发生什么]

## 实际行为
[描述实际发生了什么]

## 错误信息
[粘贴完整错误信息]

## 最小复现步骤
1. [步骤1]
2. [步骤2]
3. [步骤3]

## 我已尝试
- [尝试1]
- [尝试2]

请分析原因并给出解决方案。
```

### 视觉问题

```
## 问题描述
[描述视觉问题]

## 期望效果
[描述应该是什么样]

## 当前效果
[描述现在是什么样，可附截图]

## 相关代码
[粘贴相关 CSS/组件代码]
```

### 性能问题

```
页面/功能运行缓慢，请帮我分析：

## 症状
- [描述卡顿情况]
- [描述响应时间]

## 相关代码
[粘贴可能有问题的代码]

请：
1. 分析性能瓶颈
2. 提供优化方案
3. 解释优化原理
```

## 重构类

### 代码拆分

```
以下代码文件太大了，请帮我拆分成多个模块：

[粘贴代码]

要求：
1. 按职责拆分
2. 保持功能不变
3. 确保模块间低耦合
4. 添加必要的导入导出
```

### 提取公共逻辑

```
以下几处代码有重复逻辑，请提取为公共函数：

## 代码1
[粘贴代码]

## 代码2
[粘贴代码]

要求：
1. 提取公共部分
2. 保持各处功能不变
3. 函数命名要清晰
```

## 高级技巧

### 触发深度思考

```
Think as long as needed to get this right, I am not in a hurry.
What matters is that you follow precisely what I ask you and execute it perfectly.
Ask me questions if I am not precise enough.
```

### Claude 专用深度思考

```
think < think hard < think harder < ultrathink
```

示例：
```
ultrathink: 分析这个架构设计的优缺点，考虑扩展性、维护性、性能。
```

### 限定修改范围

```
## 你可以修改
- src/components/Button.tsx
- src/styles/button.css

## 你不能修改
- src/lib/api.ts
- package.json
- 任何配置文件

## 任务
[描述任务]
```

### 分步执行复杂任务

```
这是一个复杂任务，请分步执行：

1. 先分析现有代码结构
2. 提出修改方案，等我确认
3. 确认后再执行修改
4. 每完成一步都暂停，等我验证

任务：[描述任务]
```
