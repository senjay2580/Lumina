# Vibe Coding 入门指南

## 什么是 Vibe Coding？

> "There's a new kind of coding I call 'vibe coding', where you fully give in to the vibes, embrace exponentials, and forget that the code even exists."
> — Andrej Karpathy, 2025

Vibe Coding 是 2025 年最火的编程方式（Collins 词典年度词汇）。它的核心理念是：

- **用自然语言描述需求**，而不是手写代码
- **让 AI 生成代码**，你来验收和调整
- **快速迭代**，做出能用的东西比代码完美更重要

简单说：**你负责想法，AI 负责实现。**

---

## 核心理念：道

### 基本原则

1. **凡是 AI 能做的，就不要人工做**
2. **一切问题问 AI**
3. **目的主导**：开发过程中的一切动作围绕"目的"展开
4. **上下文是第一性要素**：垃圾进，垃圾出
5. **系统性思考**：实体、链接、功能/目的三个维度

### 思维方式

- **先结构，后代码**：一定要规划好框架，不然后面技术债还不完
- **奥卡姆剃刀**：如无必要，勿增代码
- **帕累托法则**：关注重要的那 20%
- **逆向思考**：先明确需求，从需求逆向构建代码
- **专注**：极致的专注可以击穿代码，一次只做一件事

### 关键原则

> **规划就是一切。谨慎让 AI 自主规划，否则你的代码库会变成一团无法管理的乱麻。**

## 这门课会让你获得什么？

### 你不会获得

- 精通算法和数据结构
- 成为框架源码专家
- 大厂面试通关能力

### 你会获得

- ✅ 独立交付产品的能力
- ✅ 与 AI 高效协作的方法
- ✅ 把想法变成现实的执行力

## 适合谁学习？

| 你是谁 | 推荐路径 | 理由 |
|--------|----------|------|
| 设计师 / 产品经理 | 基础篇 | 零代码基础也能做出可运行的原型 |
| 文科生 / 跨专业 | 基础篇 | 从最基础的概念开始，循序渐进 |
| 前端开发者 | 进阶篇 | 扩展后端能力，成为全栈工程师 |
| 后端开发者 | 进阶篇 | 了解现代前端生态 |
| 创业者 / 独立开发者 | 两者皆可 | 快速搭建 MVP，独立完成产品 |
| 想提升效率的开发者 | 全部 | 系统学习 AI 辅助开发工作流 |

## 推荐工具

### AI 模型 & 服务

**第一梯队（复杂任务首选）**
- Claude Opus 4.5 / Sonnet 4.5
- GPT-5 Codex (high/xhigh)

**第二梯队**
- Gemini 2.5/3.0 Pro
- Kimi K2
- GLM 4.6

### 开发环境

- **Cursor**：AI 原生 IDE，已占领用户心智
- **Claude Code**：Claude 官方 CLI 工具
- **Codex CLI**：OpenAI 官方 CLI 工具
- **Kiro**：免费使用 Claude Opus 4.5
- **VS Code**：配合 Local History 插件

### 辅助工具

- **Warp**：集成 AI 的现代化终端
- **Mermaid Chart**：文本转架构图
- **NotebookLM**：AI 解读资料
